# Issue tracker: Linear

Issues and PRDs for this repo live in Linear, in workspace `arka-agent`. Treat Linear as the source of truth for planning and execution artifacts.

## Conventions

- Create issues with clear, specific titles.
- Use concise problem statements that describe the user or system impact.
- Include explicit acceptance criteria for implementation-facing tickets.
- When creating PRDs or derived artifacts, link them back to the originating Linear issue.
- Preserve traceability between planning artifacts, implementation work, and follow-up tasks.
- Prefer standard artifact hygiene: clear scope, concrete outcomes, and minimal ambiguity.

## When a skill says "publish to the issue tracker"

Create or update the relevant item in Linear in workspace `arka-agent`, using best-practice issue structure and linking derived artifacts back to the source issue.

## When a skill says "fetch the relevant ticket"

Read the referenced Linear issue and treat its current title, description, status, and linked context as the authoritative source.
